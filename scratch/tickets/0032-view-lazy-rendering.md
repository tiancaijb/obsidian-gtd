# 0032 — View Lazy Rendering

## 目标
优化 AgendaView 的渲染策略：只渲染用户可见的分组，延迟渲染未来分组。

## 背景
当前 `AgendaView.renderGroups()` 一次性渲染所有任务分组（今天、本周、本月、未来）。当 GTD 文件较多时，未来分组可能包含大量任务，但用户很少立即滚动到最底部查看。一次性渲染所有分组增加了首次打开视图的延迟。

## 方案
- 分组优先级渲染：
  - "今天"分组 → 立即渲染（用户最常看）
  - "本周"分组 → 立即渲染（第二常用）
  - "本月"分组 → 微延迟渲染（requestAnimationFrame 或 setTimeout(0)）
  - "未来"分组 → 延迟渲染（用户滚动到附近时渲染，或 500ms 后）
- 如果用户快速关闭视图，取消未完成的延迟渲染任务
- 不引入虚拟滚动库——保持简单，只延迟渲染时机

## 任务清单
- [ ] 修改 `AgendaView.renderGroups()` 实现分层渲染
- [ ] 使用 `requestAnimationFrame` 或 `setTimeout` 延迟低优先级分组
- [ ] 在 `onunload()` 或 `onClose()` 中清理未完成的延迟任务
- [ ] 确认 `npm run build + lint + test` 通过

## 产出标准
- "今天"和"本周"分组立即渲染
- "本月"分组在下一帧渲染
- "未来"分组延迟渲染（不阻塞首次视图打开）
- 视图关闭时所有待处理的延迟任务被清理
- 功能行为不变（用户最终看到所有分组）
